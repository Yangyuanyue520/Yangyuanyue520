This is what I learned in class 280 at the University of Saskatoon
#There are several List Classes
##1.Node classes:
Each one stores an object from generic class I.
- LinkedNode
- BilinkedNode
##2.List classes:
Lists are collections of nodes.
- LinkedList
- BilinkedList
##3.Iterator classes:
LinkedIterator280 and BilinkedIterator280 implement the linear iterator interfaces.
- LinkedLiterator
- BilinkedIterator
##4.Iterator/ cursor interfaces:
An interface is a specific pattern of behaviour
- LinearIterator
- boolean before()
- void goBefore()
- boolean after()
- void goAfter()
- void goFirst()
- void goForth()
- I item()
- boolean itemExists()
- BilinearIterator
- void goLast()
- void goBack()
#Their constituent relationship is

The list classes can have cursors by implementing the same linear iterator interfaces.
- BilinkedList280 directly implements BilinearIterator280.
- LinkedList280 implements LinearIterator280
Thus the linked lists in lib280 have both cursors and iterators.
Iterators provide the same functionality as a container ADT that has a cursor, but they are separate objects from the container. This allows us to record a cursor position that is different and independent from the position recorded by the containerâ€™s internal cursor
